Secure User Authentication System

Flask-based authentication with SHA-256 password hashing and JWT session management

Python Flask SHA-256 JWT Database Management

Project Overview

Developed a production-ready authentication system using Flask that implements secure user registration, login, and session management. The system prioritizes security through industry-standard cryptographic hashing and token-based authentication, demonstrating my understanding of web security best practices.

The application handles user registration with secure password storage, implements login verification using salted hash comparison, and manages user sessions through JSON Web Tokens (JWT). This ensures that user credentials are never stored in plain text and authentication states are maintained securely across requests.

Completed in March 2025 for my Web Development course, this project showcases my ability to build secure backend systems that protect user data while providing seamless authentication experiences.

Key Features

User Registration

Secure user account creation with input validation, duplicate checking, and automated password hashing with unique salts for each user.

SHA-256 Password Hashing

Implemented cryptographic hashing using Python's hashlib with random salt generation, ensuring passwords are never stored in plain text.

JWT Session Management

Utilizes JSON Web Tokens for stateless session authentication, enabling secure API access without server-side session storage.

Secure Login Verification

Authenticates users by comparing hashed passwords with salted hash comparison, preventing timing attacks and credential exposure.

Database Integration

Persistent user data storage with proper indexing, efficient queries, and secure handling of sensitive information in the database.

User Data Management

RESTful API endpoints for creating, reading, updating, and managing user profiles with proper authorization checks and access controls.

Technical Implementation

The backend is built with Flask, Python's lightweight web framework, providing RESTful API endpoints for user operations. Password security is implemented using Python's hashlib library to generate SHA-256 hashes combined with randomly generated salts stored alongside the hashed passwords. This salt-and-hash approach ensures that even identical passwords produce different hash values.

Upon successful login, the system generates a JWT containing the user's ID and expiration time, signed with a secret key. This token is sent to the client and must be included in subsequent requests to protected endpoints. The server validates tokens by verifying the signature and checking expiration, providing stateless authentication without session storage.

The database schema includes user tables with fields for username, email, hashed password, and salt. All database queries use parameterized statements to prevent SQL injection attacks. The Flask application includes middleware for CORS handling, request validation, and error handling to provide robust and secure API responses.

Security Measures Implemented

Password Security

SHA-256 hashing with unique random salts prevents rainbow table attacks and ensures password confidentiality even if the database is compromised.

SQL Injection Prevention

Parameterized queries and input sanitization protect against SQL injection attacks across all database operations.

Token-Based Auth

JWT implementation with expiration times and signature verification prevents session hijacking and unauthorized access.

Input Validation

Comprehensive validation of all user inputs including email format checking, password strength requirements, and username constraints.

Results & Learning Outcomes

The authentication system successfully handles user registration and login with industry-standard security practices, demonstrating my ability to implement secure backend systems. This project reinforced the importance of never storing plain-text passwords and using proven cryptographic methods for user authentication.
