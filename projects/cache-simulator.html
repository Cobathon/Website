Cache Memory Simulator

Detailed cache simulator emulating load/store operations between CPU registers, cache, and RAM

C++ File I/O Data Structures Systems Programming Memory Architecture

Project Overview

Built a comprehensive cache memory simulator in C++ that accurately models the behavior of a CPU cache system, including load and store operations between CPU registers, cache memory, and RAM. This project demonstrates deep understanding of computer architecture and memory hierarchy.

The simulator implements a realistic cache model with tag management, valid and modified (dirty) bits, and block-level data storage. It handles cache hits and misses, implements write-back policies, and manages block transfers between cache and RAM to closely mimic real hardware behavior.

Completed in June 2025 for my Computer Architecture course, this project required meticulous attention to detail in implementing cache line structures, addressing logic, and ensuring cycle-accurate simulation of memory operations.

Key Features

Cache Line Data Structures

Designed comprehensive cache line structures with tag fields, valid bits, modified (dirty) bits, and block-level data storage arrays.

Load/Store Operations

Implemented CPU register to cache and cache to RAM data transfers, accurately simulating real memory access patterns and latencies.

Tag Management System

Sophisticated address parsing and tag comparison logic for determining cache hits and misses with configurable cache organization parameters.

Memory-Mapped RAM Loading

Parses hex-formatted memory files to initialize simulated RAM with realistic memory contents, supporting various test scenarios and workloads.

Write-Back Policy

Implements write-back cache policy with dirty bit tracking, minimizing memory writes and accurately modeling modern cache coherency protocols.

Performance Statistics

Tracks hit rates, miss rates, write-backs, and memory accesses to provide detailed performance analysis and cache efficiency metrics.

Technical Implementation

The cache simulator uses object-oriented design with classes representing cache lines, cache sets, and the overall cache structure. Each cache line contains fields for the tag, valid bit, modified bit, and a data block array. The addressing scheme extracts tag, set index, and block offset from memory addresses to locate data in the cache hierarchy.

For cache hits, data is read from or written to the appropriate cache line. For cache misses, the simulator implements replacement policies (like LRU) to evict cache lines when necessary. When a dirty line is evicted, it writes back the modified block to RAM before loading the new block from memory.

File I/O handles reading hex-formatted RAM initialization files, parsing them into memory structures, and supporting trace-driven simulation where sequences of load/store operations test cache behavior under various access patterns. The simulator outputs detailed logs of each operation including hits, misses, evictions, and write-backs.
