Project Promotion

Turn-based RPG with A* pathfinding AI, conditional turn-based system, and modular behavior trees

Unity C# A* Pathfinding AI Programming Game Development

Overview

Project Promotion is a strategic turn-based RPG featuring intelligent AI systems and complex game mechanics. The game showcases advanced programming techniques including A* pathfinding for enemy navigation, a Conditional Turn-Based (CTB) system using min-heap data structures for efficient turn order management, and modular behavior trees that enable flexible and maintainable character decision-making logic.

As an AI and Gameplay Programmer, I was responsible for implementing core game systems including behavior tree architectures, save/load progression systems, level progression mechanics, and cutscene controllers. The project combines my computer science background with game design skills, resulting in a technically sophisticated RPG with engaging gameplay mechanics that demonstrate both algorithmic thinking and game development expertise.

Challenges & Solutions

Key technical problems I encountered and how I solved them.

Challenge 1: Learning AI and Behavior Trees from Scratch

AI programming and behavior tree architecture were completely new concepts to me when I started this project. I needed to understand not just how to implement these systems, but also the underlying design patterns and best practices for creating maintainable, extensible AI architectures.

Solution:

Engaged in extensive research on behavior tree patterns, AI decision-making systems, and game AI architecture. Studied existing implementations and design patterns through documentation, tutorials, and academic resources. Used a trial-and-error approach with iterative prototyping, starting with simple behavior trees and gradually increasing complexity. Created test scenarios to validate each system component before integration. Documented learnings and patterns to build a solid foundation for future development.

Impact:

Successfully implemented a robust behavior tree system that supports complex AI behaviors. The modular architecture allows for easy extension and modification without breaking existing functionality. The knowledge gained enabled me to contribute effectively to other AI systems in the project.

Challenge 2: Designing Save/Load System for Complex Game State

Creating a save/load system that accurately preserves all game state including player progression, character data, level unlocks, and AI states required careful design to ensure data integrity and system performance.

Solution:

Designed a serialization system using Unity's built-in serialization capabilities combined with custom data structures. Created a centralized save manager that coordinates saving and loading across all game systems. Implemented versioning to handle save file compatibility as the game evolves. Used defensive programming to handle edge cases and corrupted save files gracefully.

Impact:

Players can reliably save and resume their progress. The system is extensible, allowing new game systems to easily integrate with the save/load functionality. Save files remain compatible across game updates through version management.

Challenge 3: Integrating Multiple Systems Without Tight Coupling

The behavior trees, cutscene controller, level progression, and save/load systems all needed to work together seamlessly without creating dependencies that would make the codebase difficult to maintain or modify.

Solution:

Implemented event-driven architecture using C# delegates and events. Systems communicate through well-defined interfaces rather than direct references. Used Unity's component system to keep systems modular and independently testable. Created clear separation of concerns where each system has a single responsibility.

Impact:

Systems can be developed and tested independently. Changes to one system don't require modifications to others. The architecture supports parallel development by multiple team members without conflicts. Debugging is easier due to clear system boundaries.

Code Examples & Breakdown

Detailed breakdown of key systems with explanations of functionality, structure, and design decisions.

Behavior Tree System - Modular Architecture

The behavior tree system uses a hierarchical node structure where composite nodes (selectors, sequences) control flow, and leaf nodes (actions, conditions) perform specific tasks. This architecture enables complex AI behaviors through composition of simple, reusable components.

Key Features:

  • Modular node-based architecture for flexible AI behavior composition
  • Composite nodes (Selector, Sequence) control decision flow
  • Leaf nodes (Action, Condition) perform specific tasks
  • State management through node return values (Success, Failure, Running)
  • Extensible design allowing easy addition of new node types
  • Clear separation between behavior logic and game state

Save/Load System - Serialization Architecture

The save/load system uses Unity's serialization capabilities combined with custom data structures to preserve game state. A centralized SaveManager coordinates saving and loading across all game systems, ensuring data consistency and integrity.

Key Features:

  • Centralized SaveManager coordinating all save/load operations
  • Serializable data structures for game state preservation
  • Version management for save file compatibility
  • Defensive programming handling corrupted or missing save files
  • Event-driven notifications for systems that need to save/load data
  • Efficient binary serialization for performance

Cutscene Controller - Event-Driven System

The cutscene controller uses an event-driven architecture to manage narrative sequences and gameplay transitions. It integrates seamlessly with other systems through well-defined interfaces, allowing cutscenes to trigger game events and respond to player actions.

Key Features:

  • Event-driven architecture for loose coupling with other systems
  • Timeline-based cutscene management
  • Integration with gameplay systems through events
  • Support for branching narratives and player choices
  • Seamless transitions between cutscenes and gameplay
  • Extensible design for adding new cutscene types

A* Pathfinding - Optimal Path Calculation

The A* pathfinding implementation uses a priority queue (min-heap) to efficiently find optimal paths through the game world. The algorithm considers terrain costs and movement constraints to calculate the best route for AI characters.

Key Features:

  • Optimal pathfinding using A* algorithm with heuristic estimation
  • Efficient data structures (min-heap) for priority queue management
  • Terrain-aware pathfinding considering movement costs
  • Performance optimization for real-time path calculation
  • Support for dynamic obstacles and changing terrain
  • Path smoothing for natural-looking movement
